Bacon styleguide

Coding style

Variables

All global settings for typography, document, colour palette etc are stored in settings/_settings.core.scss. Component-specific settings should be set at the top of that component's file.

Colour palette

The global colour palette is contained within a Sass map called $global-colors. Colours within this map are grouped together according to their usage or subject. Each new colour group must, at a minimum, have a base colour entry. You can then add as many others as you wish.

$global-colors: (
  brand: (
    base: #1796e1,
    secondary: #1bb2fe,
    tertiary: #0377c0,
    accent: #fada00
  ),
  grayscale: (
    base: #000,
    dark: #3c3c3c,
    mid: #676c71,
    light: #b1b9c3,
    off-white: #ededed
  )
);

To access these colours, use the color() function.
color(notifications) will give you the default base colour from the notifications group.
color(notifications, warning) will give you the warning colour from the notifications group.

Advantage of using colour groups:

  • Enforces consistency (via use of color() function)
  • Easy to identify where a brand colour is being used (by searching for the color() function)
  • Allows easy repitition where programatic utility classes are required (use a Sass @for on the map)

Whitespace

Bacon is liberal with whitespace to aid readability. The general rule is two blank lines between blocks and one blank line inside blocks.

/* ==========================================================================
   Buttons
   ========================================================================== */


// Component variables

$button-padding-y: 1em !default;
$button-padding-x: 4em !default;


/**
 * Standard button component.
 */

.button {
  display: inline-block;
  padding: $button-padding-y $button-padding-x;

  &:active {
    background-color: #000;
    color: #fff;
  }
}

It is important to follow these guidelines to ensure document flow but always prefer to put in one blank line too many than one too few should you be unsure.

Commenting

Bacon follows the commenting style outlined in Harry Roberts' CSS Guidelines plus a couple of Sass-y additions. Broadly, use C-style comment blocks to document code and for anything Sass-related, use inline double-slash comments (//).

There are two levels of C-style comments; one for file headers and one for specific blocks of CSS.

File header

Only used at the very top of files. Every .scss file should open with one of these comments giving a title and an additional description if necessary.

/* ==========================================================================
   Buttons
   ========================================================================== */

Block comment

Used for individual blocks of CSS within the same file.

/**
 * Link button variant.
 */

Block should have a title and then, if required, a description separated by one blank line (with an asterisk). Following CSS Guidelines, use footnotes to describe anything unusual or noteworthy in the accompanying block. Ensure that every line in these blocks has a full stop at the end.

/**
 * Standard button component.
 *
 * 1. Line different size buttons up vertically.
 * 2. Force all buttons to appear clickable
 */

.button {
  display: inline-block;
  margin: 0;
  vertical-align: middle; // [1]
  text-align: center;
  cursor: pointer; // [2]
}

Sass double-slash comments

For any Sass-specific comments (which shouldn't be included in the compiled output) use double-slash quotes.

// Component variables

$button-padding-y: 1em !default;
$button-padding-x: 4em !default;

Choice inputs: checkboxes and radios

Checkboxes and radios share styles via the .choice-input class (and child classes). They are then modified using their own .checkbox and .radio classes.

Always wrap the <input> and sibling elements in a <label> because the :checked attribute is combined with an adjacent sibling selector to style the indicator box/circle.
